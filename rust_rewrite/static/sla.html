Service Level Agreement (SLA)

Version: 1.0
Date: 14/6-2025

1. Definitions

  • Service: The Sauron search API and web UI running at https://sauron.tolana.dev
  • Uptime: The percentage of time the Service is available to respond to valid HTTP requests.
  • Downtime: Any period when the Service fails to respond within the agreed performance targets.

2. Service Availability

  • Target Uptime: 99.5% per calendar month
  • Measurement: Excluding scheduled maintenance windows (see §5), calculated as:
    (Total Minutes – Downtime Minutes) / Total Minutes × 100

3. Performance

  • API Response Time:
    • 95th percentile of /api/* endpoints ≤ 500 ms
    • 99th percentile of /api/* endpoints ≤ 1 s
  • Metrics: Monitored via Prometheus and Grafana dashboards; alerts fire if thresholds are breached for 5 minutes.

    • Note: these figures are goals, actual performance may vary.

4. Support & Incident Response

Severity Description Resolution Target
P1 (Critical) Service down or major functionality broken (e.g. search, register) 24 hours
P2 (High) Significant degradation (slow responses, partial feature failure) 48 hours
P3 (Normal) Minor issues (UI glitches, non-critical errors) 72 hours

5. Maintenance Windows

  • Regular maintenance (database migrations, deployment) will occur regularly, and shouldn't last more than a couple minutes.
  • Planned downtime during maintenance does not count against uptime.

6. Exclusions

  • Force majeure events (e.g. natural disasters, major internet outages).
  • Third-party outages (e.g. external API providers).
  • Customer network issues or misconfiguration.

7. Reporting & Review

  • Monthly uptime and performance reports generated from Prometheus metrics.
