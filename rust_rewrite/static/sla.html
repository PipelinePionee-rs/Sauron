Service Level Agreement (SLA)

Version: 1.0
Date: 14/6-2025

1. Service Scope

  • Service: The Sauron search API and web UI at https://sauron.tolana.dev
  • Features: User registration, authentication, search functionality, and web-based results display.
  • Processes: Data indexing, query processing, user management, and monitoring.

2. Performance Metrics

  • Target Uptime: 99.5% per calendar month (excluding scheduled maintenance).
  • API Response Time:
    • 95th percentile of /api/* endpoints ≤ 500 ms
    • 99th percentile of /api/* endpoints ≤ 1 s
  • Measurement: Metrics monitored via Prometheus and Grafana; alerts fire if thresholds are breached for 5 minutes.
Note: These figures are goals; actual performance may vary.

3. Response and Resolution

Severity Description Resolution Target
P1 (Critical) Service down or major functionality broken (e.g. search, register) 24 hours
P2 (High) Significant degradation (slow responses, partial feature failure) 48 hours
P3 (Normal) Minor issues (UI glitches, non-critical errors) 72 hours

4. Security Measures

  • Encrypted connections (HTTPS/TLS) for all client-server communication.
  • Regular security updates and vulnerability patching.
  • Role-based access control for sensitive operations.
  • Continuous monitoring and alerting for suspicious activity.

5. Compliance Standards

  • GDPR compliance for user data privacy and handling.
  • Adherence to industry best practices for data protection and security.

6. Maintenance Windows

  • Regular maintenance (database migrations, deployment) will occur regularly, and shouldn't last more than a couple minutes.
  • Planned downtime during maintenance does not count against uptime.

7. Exclusions

  • Force majeure events (e.g. natural disasters, major internet outages).
  • Third-party outages (e.g. external API providers).
  • Customer network issues or misconfiguration.
